# LROM_Project — Agent Orientation

Read this before changing anything in this repository.

## Naming laws (do not deviate)

- **Package versions**: 1.x = wavefunction emulation (current, `lrom/`),
  2.0 = cross sections (parked in `lrom_legacy/v2_0`, needs fixes),
  3.0 = global KD emulator (future). Minor digits are fixes within a
  milestone; the next milestone is never a minor bump.
- **Benchmark notebooks are named after the legacy notebook they recreate**:
  `benchmark_02.ipynb` recreates the archived
  `scientific_archive/legacy_code/Legacy_benchmark/notebooks/02_lrom_method_walkthrough.ipynb`.
  A future cross-section benchmark is `benchmark_03.ipynb`.
- Full rules: `docs/VERSIONING.md`.

## Current state (2026-07-13)

- **`lrom` is ONE file** (`lrom/__init__.py`, version 1.1.0): a single
  `LROM` object exposing `.sampling`, `.train`, `.predict`, plus everything
  it needs (potentials, config, sampling designs, centered SVD basis,
  predictors, RF-LROM core, ROSE FOM boundary, shared-basis ROSE RBM,
  portable artifacts). **Do not split it back into modules** — the single
  file is a deliberate user requirement.
- Version 1 physics is being re-verified against `benchmark_02`; that is why
  2.0 was parked. `lrom_legacy/v2_0`'s docstring lists its open fixes
  (spin-orbit-blind predictors, noisy CAT timing).
- `notebooks/01_rbm_vs_lrom_single_wavefunction.ipynb` is generated by
  `_agents/scripts/generate_notebook01.py` — edit the generator, not the
  notebook JSON.
- 2026-07-12 incident: iCloud eviction destroyed the previous local `.git`
  (history reconstructed in `_agents/RECOVERED_HISTORY.md`). **Push to
  GitHub after every milestone** — the user has durably authorized pushes.
  Merge everything to `main`; no side branches or temp storage.

## Layout

- `lrom/`, `lrom_legacy/`, `notebooks/`, `docs/`, `tests/`,
  `scientific_archive/`, `outputs/` — researcher-facing.
- `_agents/` — agent working area (plans, specs, backlog, validation
  notebooks, generation scripts). See `_agents/README.md`. Put new agent
  plans/specs/scratch there, not in the root or `docs/`.
- `scientific_archive/` is read-only reference material. Never modify it.

## Deliberate decisions — do not "fix"

- benchmark_02's ROSE comparison runs the public `rose` package natively,
  inline in notebook cells, with ROSE's free-reference basis while the LROM
  uses a central-reference basis. This apples-to-oranges setup is
  intentional; coefficients are compared only within one convention.
- No plotting wrapper functions in benchmark notebooks; all Matplotlib
  inline. Comments stay simple, aimed at a physics-literate reader.
- `DIFFERENCE_FLOOR = 1e-5` display floor in difference figures
  (display-only clipping).
- EIM `training_info` bounds include testing rows; changing this is a
  scientific decision reserved to the user.

## Workflow expectations

- Run `python -m pytest -q` before and after changes; keep the suite green.
- Execute notebooks end-to-end before claiming they work
  (superpowers `verification-before-completion`).
- The workspace-level memory protocol (`../CLAUDE.md`,
  `../_memory/Context/active-state.md`) applies here too.
