# LROM_Project — Agent Orientation

Read this before changing anything in this repository.

## Naming laws (do not deviate)

- **Package versions**: 1.2 = current public `lrom/`, implemented in the
  authoritative single file `lrom_legacy/v1_2/__init__.py`; 2.0 = parked
  future shell in `lrom_legacy/v2_0` (wavefunctions + cross sections);
  3.0 = global KD emulator (future). Minor digits are fixes within a
  milestone; the next milestone is never a minor bump.
- **Benchmarks live under `notebooks/benchmark_notebooks/<version>/` and are
  named after the notebook they recreate**: `1.0/benchmark_02.ipynb` (legacy
  02), `2.0/benchmark_03.ipynb` (legacy 03), `2.0/benchmark_01.ipynb` (our
  notebook 01, plus the 2.0-vs-1.2 comparison).
- Full rules: `docs/VERSIONING.md`.

## Layout

- `lrom/`, `lrom_legacy/`, `notebooks/`, `docs/`, `tests/`,
  `scientific_archive/`, `outputs/` — researcher-facing.
- `.agents/` — agent working area (plans, specs, backlog, validation
  notebooks). Dot-prefixed deliberately: the GitHub repo is shared and the
  user wants AI-only material out of the way, but it stays versioned so
  agent context survives local file loss. Put new agent plans/specs/scratch
  there, not in the root or `docs/`.
- `tools/` — notebook generators exercised by the tests (importable, so it
  cannot live inside the dot-directory).
- `scientific_archive/` is read-only reference material. Never modify it.

## Deliberate decisions — do not "fix"

- **ROSE never benchmarks inside the package** (since 1.2.0): the engine
  uses ROSE only as the full-order solver; every ROSE emulator comparison
  is constructed in the notebooks with the public `nuclear-rose` package
  (notebook 01 shares the package basis explicitly; benchmark_02 uses
  ROSE's free-reference convention).
- **Exact ROSE high-fidelity boundary:** package `sampling()` constructs
  `InteractionSpace` and supports `high_fidelity_solver="runge_kutta"`.
  It does not construct an EIM.
- **LS baseline is opt-in:** RF-LROM training coordinates remain required,
  but a testing-set least-squares reconstruction is called explicitly with
  `least_squares_baseline()`.
- **Public object rigidity:** `LROM(...)`, `sampling()`, `train()`,
  `predict()`, `save()`, and `load()` keep their lifecycle meaning. A major
  restructure requires user approval.

- benchmark_02's ROSE comparison runs the public `rose` package natively,
  inline in notebook cells, with ROSE's free-reference basis while the LROM
  uses a central-reference basis. This apples-to-oranges setup is
  intentional; coefficients are compared only within one convention.
- No plotting wrapper functions in benchmark notebooks; all Matplotlib
  inline. Comments stay simple, aimed at a physics-literate reader.
- `DIFFERENCE_FLOOR = 1e-5` display floor in difference figures
  (display-only clipping).
- ROSE benchmark EIM construction belongs visibly in notebook methodology;
  it is not owned by package sampling.
