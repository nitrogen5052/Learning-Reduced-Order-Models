## Goal

Extend the single-file `lrom` package to version **2.0** so it can produce
paper Notebook 2 per the Paper Results Map (distilled in
`.agents/backlog/paper-results-roadmap.md`):

- ROSE and LROM compared at the **cross-section level**, varying **all**
  optical-potential parameters from the ROSE paper.
- Identical training samples for both methods; same basis size.
- CAT-plot space comparison across basis and operator sizes.
- No linear-LROM baseline in final figures.
- Optional: potential rainbows + predictor locations for representative l.
- Explore exporting a trained emulator as an interactive HTML application
  (deferred; last).

## Architecture rules (locked by the user)

1. **The package stays ONE file** (`lrom/__init__.py`); 2.0 features extend
   the same `LROM` object (`observable="cross_section"`,
   `angles_degrees=...`) — no new modules.
2. **No ROSE benchmarking inside the package.** ROSE is used only as the
   full-order solver. All ROSE emulator comparisons are constructed in the
   notebooks with the public `nuclear-rose` package (2026-07-13 rule; the
   engine was already stripped for 1.2.0).
3. The wavefunction (1.x) API must keep working unchanged — notebook 01 and
   benchmark_02 are regression anchors.
4. Version bumps to 2.0.0 when cross-section prediction lands and its
   benchmark passes.

## What to reuse from the parked attempt (`lrom_legacy/v2_0`)

Salvage, with the known fixes:
- `full_woods-saxon` potential with ROSE `KD_simple` sign conventions
  (positive `Wv`/`Wd`, absorptive imaginary terms) and `Rso` derived from
  the target mass (`full_woods_saxon_central(target_a=...)`).
- Spin-orbit split channel handling (`(l, spin)` keys) and the
  S-matrix-from-coefficients + `calculate_xs` prediction path.
- The corrected `sph_harm` shim (angle order) — cross sections evaluate
  spherical harmonics, so this matters now.

## Known flaws that MUST be fixed in the new implementation

1. **Spin-orbit-blind predictors** (the killer): potential predictors were
   built from the central (spin-orbit-free) potential and carried no
   `Vso`/`Rso`/`aso` information → LROM cross-section median error ~0.34 vs
   ROSE(8,12) 1.7e-3. Design predictors that see every varied parameter,
   e.g. maxvol points drawn from the stacked central + spin-orbit radial
   profiles, or per-channel effective potentials.
2. **CAT timing methodology**: batch-minus-single differencing produced a
   negative per-sample time. Use min-of-N-repeats for both the batch and
   the setup measurement, and report the online stage only.
3. The S-matrix assembly (`ScatteringAmplitudeEmulator`) was rebuilt on
   every `predict` call (~0.8 s). Cache it on the trained object.
