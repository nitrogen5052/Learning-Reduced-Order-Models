## Root-Cause Findings

The LS, LROM, ROSE, high-fidelity, case-ID, and parameter arrays already use the same ordered testing design. The scientific problem is selection, not array ordering.

The current `Vv` selector removes the exact center and then takes the middle remaining array index. That selects `test-0021`, only 0.854 MeV from the central depth, or five percent of the testing half-range. It is technically noncentral but remains an unusually easy near-reference case.

The current `ws_3` selector chooses the median LROM error. Its selected row is not central and does not overlap training, but the criterion favors LROM alone. In the controlled diagnostic its LS and LROM relative errors are both about `5.3e-3`, while its ROSE error is about `5.27e-2`.

## Matched Three-Case Methodology

Both studies use the same method-neutral selection procedure.

1. Build one table of LS, LROM, and ROSE relative L2 errors for every testing row.
2. Scale each active parameter displacement by that parameter's testing half-range and take the Euclidean norm. Exclude an exact central row, any exact training-row overlap, and cases with normalized distance below 0.25.
3. Convert each method's candidate errors to ordinal percentile ranks with NumPy sorting. Ranks give LS, LROM, and ROSE equal influence even when their raw error scales differ.
4. Average the three method ranks into one combined difficulty score.
5. Select distinct candidates nearest the 25th, 50th, and 75th percentiles of combined difficulty.
6. Assert that each selected case ID and parameter dictionary map back to the same testing row used by the high-fidelity, LS, LROM, and ROSE arrays.
7. Print the three case IDs, physical parameters, normalized central distance, and all three relative errors.

Each study then uses one 2-by-3 figure. Columns are the lower, median, and upper combined-difficulty cases. The top row shows high fidelity, LS, LROM, and ROSE wavefunctions. The bottom row shows their pointwise absolute errors on a log axis. No displayed case is central or an exact training-row overlap.

## Coordinate Figures

All existing coordinate-difference figures are deleted. ROSE coordinates are not subtracted from LROM/LS coordinates because ROSE uses a free-reference basis while LROM and LS use the central-reference basis.

Each study receives one combined coordinate figure with method rows and coordinate columns:

- rows: LS, LROM, ROSE;
- columns: retained coordinates 1 through 4;
- one-parameter study: horizontal axis `Vv`;
- three-parameter study: horizontal axis `Vv`, with testing points colored by `Rv` on one shared color scale;
- LS/LROM axes use `a_j`; ROSE axes use `c_j` to keep the basis convention visible;
- training coordinates are shown as small black open circles, while testing coordinates use the method styling in the `Vv` study and the shared `Rv` color scale in the three-parameter study;
- no coefficient-difference panels are included.

This is one figure per study, places every method over the same physical testing rows, and does not imply equality between the two coordinate conventions.
