## Confirmed Root Cause

The installed ROSE source documents `Basis.phi_0` and `CustomBasis.phi_0` as the free solution with no interaction. `ReducedBasisEmulator.initialize_emulator()` uses that array in:

- the inhomogeneous Galerkin terms `b_1`, `b_2`, and `b_3`;
- reconstructed wavefunctions through `basis.phi_hat()`;
- matching-radius values and derivatives through `phi_components`.

Notebook 01 currently passes the central-potential high-fidelity solution and then overwrites the custom basis reference and vectors with LROM's central-reference basis. That is not ROSE's native affine convention and invalidates the existing shared-coordinate interpretation.

## Selected Method

ROSE and LROM will use the same v1.2 high-fidelity training snapshots, training parameter rows, mesh, and retained basis size `K = 4`. They will use their own correct affine conventions:

- **LROM:** central high-fidelity solution plus four SVD vectors of central-centered snapshots.
- **ROSE:** analytical free solution plus four SVD vectors of free-centered snapshots.

For each study, construct the ROSE basis with `rose.basis.CustomBasis`:

1. evaluate the analytical free solution on `samples.mesh.rho` using ROSE's public free-solution routine point by point;
2. pass the v1.2 training snapshots as the `solutions` matrix;
3. set `subtract_phi0=True`, `use_svd=True`, `center=False`, and `scale=False`;
4. retain `n_basis=4`;
5. do not overwrite `basis.phi_0` or `basis.vectors` afterward;
6. create `ReducedBasisEmulator` with the v1.2 interaction and solver state.

This is an equal-data, equal-rank comparison, but not a same-coordinate comparison. ROSE and LROM coefficient values therefore must not be overlaid or subtracted from one another.

## Singular-Value Diagnostics

No package change is allowed, and v1.2 stores only the retained singular values in `BasisState`. The notebook will therefore compute the diagnostic spectrum directly from the authoritative snapshots:

- LROM spectrum: SVD of `training_wavefunctions - central_wavefunction`;
- ROSE spectrum: use `CustomBasis.singular_values`, obtained from free-centered snapshots.

Every wavefunction basis-vector display becomes a two-panel figure:

1. retained basis vectors versus physical radius `r [fm]`;
2. normalized singular values `sigma_j / sigma_1` on a log axis, with the retained cutoff `K = 4` marked.

These spectra are diagnostics only. They do not replace package state and do not change training or inference.
